コマンド

init: gitリポジトリを作成する

  • git init

clone: 既存のリポジトリからローカルリポジトリを作成する

  • git clone git@github.com:anyone/somerepository.git
  • git clone ~/existing/repo ~/new/repo

add: git-add - 索引(index)にファイルの内容を追加する

rm: ワーキング ツリーとインデックスからファイルを消します．

commit: インデックスの変更をリポジトリにコミットする

  • git commit

merge: 現在のbranchに <otherbranch>をマージする

  • git merge <otherbranch> : 現在のbranchに <otherbranch>をマージする
  • git merge <otherbranch> --squash : コミットを1つに固める。コミットメッセージは最新のが使われる
  • git merge <otherbranch> --squash --no-commit : コミットを1つに固める。コミットメッセージは後でcommitした時に決める

pull: 他のリポジトリまたはローカル ブランチから取得しマージします．

push: 公開リポジトリに自分のリポジトリの内容を送信する

status: 変更が加えられたファイルを表示する

  • git status

revert: すでに存在しているコミットを打ち消すコミットを作成する．

branch: HEADから<newbranch>を作る

  • git branch <newbranch> : HEADから<newbranch>を作る
  • git branch <newbranch> <oldbranch> : <oldbranch>から<newbranch>を作る
  • git branch -m <oldbranch> <newbranch> : branch名を変える
  • git branch -d <branch> : branchを削除する

rebase: ブランチの派生元を変更する

  • git rebase <upstream> : <upstream>に枝を繋ぎ直す
  • git rebase -i <upstream> : interactive（対話モード）。<upstream> 〜 HEADまでを歴史改変する(<upstream>は含まない)
  • git rebase --continue : rebaseを再開する
  • git rebase --abort : rebaseを中断する

log: コミットログを見る

  • git log : コミットログを見る
  • git log -- <some/file/path> : 特定のファイルのコミットログを見る

cherry-pick: 別のブランチのコミットの変更内容を現在のブランチに取り込む

  • git cherry-pick <commit> : 別のブランチのコミットの変更内容を現在のブランチに取り込む

fetch: 他のリポジトリから変更を取得する

  • git fetch : 他のリポジトリから変更を取得する
  • git fetch --all : 全てのリポジトリから変更を取得する

git config alias: コマンドのエイリアスを設定する

  • git config --global alias.co checkout：checkout コマンドを co で使えるようにする
  • git config --global alias.ci commit：commit コマンドを ci で使えるようにする
  • git config --global alias.st status：status コマンドを st で使えるようにする

単語

revision: Gitでは特定のコミットはハッシュ値(SHA-1)が使われる

branch: コミットのつながり
